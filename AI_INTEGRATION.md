# AI Integration Documentation

## Overview

This document explains how the AI extraction feature works in the Context Assistant application, including the data flow from frontend to backend and the transformation logic.

## Architecture

```
Frontend (React)
    ↓
    POST /extract { text: "..." }
    ↓
Backend (Express)
    ↓
    AI Service (Ollama DeepSeek-R1:1.5b)
    ↓
    Transform AI Result → Frontend Format
    ↓
    Return { interests, importantDates, places, notes }
    ↓
Frontend processes arrays safely
```

## Data Flow

### 1. Frontend Request (App.tsx)

```typescript
const response = await fetch("http://localhost:5000/extract", {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({ text, messageId: message.id }),
});

const data = await response.json();
// data = { interests: [], importantDates: [], places: [], notes: [] }
```

### 2. Backend Processing (index.ts)

The `/extract` endpoint:
- Validates input text
- Calls AI service for extraction
- Transforms AI result to frontend format
- Returns guaranteed arrays (even on error)

```typescript
// Returns this format ALWAYS:
{
  interests: [{ value: "sushi" }],
  importantDates: [{ value: "April 5" }],
  places: [{ value: "Seattle" }],
  notes: [{ value: "general info" }]
}
```

### 3. AI Service (ai.service.ts)

**Function: `extractFactsFromText(text: string)`**

Sends text to Ollama AI with structured prompt requesting JSON output:

```json
{
  "intent": "add_contact",
  "payload": {
    "contact_name": "Abby",
    "facts": [
      { "type": "interest", "value": "sushi", "source_text": "I like sushi" },
      { "type": "important_date", "value": "April 5", "source_text": "birthday April 5" }
    ]
  },
  "confidence": 1.0,
  "needs_clarification": false
}
```

**Function: `transformToFrontendFormat(aiResult: AIResult)`**

Transforms AI result into frontend-ready format:

```typescript
// Input: AI result with facts array
// Output: Categorized arrays
{
  interests: [],      // type === "interest"
  importantDates: [], // type === "important_date"
  places: [],         // type === "place"
  notes: []           // type === "note" or unknown types
}
```

**Key Features:**
- ✅ Always returns all 4 arrays (never undefined)
- ✅ Handles missing/malformed AI responses gracefully
- ✅ Unknown types fallback to "notes"
- ✅ Trims whitespace from values
- ✅ Skips empty/invalid facts

## Type Definitions

### Backend Types (ai.types.ts)

```typescript
export interface AIFact {
  type: "interest" | "important_date" | "place" | "note";
  value: string;
  source_text?: string;
}

export interface AIResult {
  intent: string;
  payload: {
    contact_name?: string;
    facts: AIFact[];
  };
  confidence: number;
  needs_clarification: boolean;
}

export interface FrontendExtractedData {
  interests: Array<{ value: string }>;
  importantDates: Array<{ value: string }>;
  places: Array<{ value: string }>;
  notes: Array<{ value: string }>;
}
```

### Frontend Types (person.ts)

```typescript
export type InfoItem = {
  id: string;        // Generated by frontend
  value: string;     // From backend
  messageId: string; // Message reference
};
```

## Error Handling

### Backend Error Response

Even on errors, the backend returns safe empty arrays:

```json
{
  "error": "AI extraction failed",
  "message": "Details...",
  "interests": [],
  "importantDates": [],
  "places": [],
  "notes": []
}
```

### Frontend Safe Array Handling

```typescript
const safeInterests = Array.isArray(data?.interests) ? data.interests : [];
const safeDates = Array.isArray(data?.importantDates) ? data.importantDates : [];
// ... safely map over arrays
```

## AI Prompt Engineering

The system prompt instructs the AI to:

1. Return ONLY valid JSON (no markdown, no explanation)
2. Use exact structure with type-safe categories
3. Extract ALL facts thoroughly
4. Categorize by type:
   - **interest**: hobbies, likes, preferences
   - **important_date**: birthdays, anniversaries, deadlines
   - **place**: locations, addresses, venues
   - **note**: general information
